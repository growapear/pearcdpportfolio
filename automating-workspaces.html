Automating Flexible Workspaces

A generative design tool that automates workspace layouts based on users' schedules, roles, and task types.

View Project Website →

Overview

Automating Flexible Workspaces is a generative design project that addresses a growing need: how to make spatial layouts adapt to dynamic, hybrid schedules. Rather than designing one-size-fits-all office environments, this tool generates real-time layout configurations based on who is present, when, and what they need to do. Users input their daily work schedules, preferred locations, and task types (e.g., focused work, collaboration, heads-down time). The system then creates an optimized workspace arrangement that minimizes unnecessary movement, maximizes overlap among collaborators, and respects focus zones. Built within Rhino/Grasshopper using Python, the system integrates a shortest-path algorithm and a custom visualization layer that encodes task difficulty through color and node size. This project reframes layout not as a static plan, but as a time-responsive system. It imagines the office as a living network—capable of rearranging itself based on behavioral patterns, not just architectural boundaries. Ultimately, the tool empowers both designers and occupants to better align space with daily rhythms, bridging human-centered scheduling with architectural logic.

Process

The project began by identifying a core problem in hybrid workspaces: space is often underutilized or poorly matched to task needs. I started by collecting example schedules and translating them into a flexible data structure. Using Rhino and Grasshopper, I scripted a Python-based layout generator that interprets each individual's presence, role, and task priority. The tool uses a shortest-path algorithm to minimize total distance traveled while encouraging proximity between collaborators. It also visualizes task density by using larger nodes for more complex activities. Iterations focused on refining user input logic, spatial conflict detection, and the visual system used to read the generated layouts.

Technical Details

The system utilizes:

Python-Scripted Spatial Generator

Custom code embedded in Grasshopper reads user schedule data and generates optimized spatial layouts

Shortest-Path Algorithm

Calculates movement efficiency and proximity clustering between collaborators

Task-Based Spatial Encoding

Visual elements like node size and color map task intensity, collaboration needs, and workspace type

Real-Time Feedback Loop

Users can adjust schedules or task weights and immediately preview the resulting spatial change
